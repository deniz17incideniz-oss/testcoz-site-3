Google AdSense Kurulum Rehberi

AdSense başvurusu onaylandıktan sonra kodları siteye eklemek için bu rehberi takip edin.

⚠️ Bu sayfa yalnızca yönetici kullanımı içindir

Bu sayfa arama motorlarına kapalıdır (noindex). Canlıya geçmeden önce bu dosyayı silin veya şifre koruması ekleyin.

1. AdSense Başvurusu Onaylandıktan Sonra

Google AdSense hesabınız onaylandığında size bir Publisher ID (ca-pub-XXXXXXXXXXXXXXXX) verilecektir. Bu ID'yi aşağıdaki adımlarda kullanacaksınız.


ca-pub-XXXXXXXXXXXXXXXX

2. Auto Ads Kodunu <head> İçine Ekleyin

Her HTML sayfasının <head> bölümüne aşağıdaki kodu ekleyin. Tüm sayfalara eklemeniz gerekir: index.html, sinif.html, konu.html, veli.html, hakkimizda.html, iletisim.html, gizlilik-politikasi.html, kullanim-sartlari.html

<!-- Google AdSense -->
<script async
  src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-XXXXXXXXXXXXXXXX"
  crossorigin="anonymous"></script>

ca-pub-XXXXXXXXXXXXXXXX yerine kendi Publisher ID'nizi yazın.

3. Ad Unit (Manuel Reklam Birimi) Ekleme

Sitedeki .ad-zone div'lerini gerçek AdSense kod bloklarıyla değiştirin. AdSense panelinden yeni bir "Ad unit" oluşturun ve aşağıdaki formatı kullanın:

<!-- Leaderboard 728x90 — Hero altı -->
<ins class="adsbygoogle"
    style="display:block"
    data-ad-client="ca-pub-XXXXXXXXXXXXXXXX"
    data-ad-slot="XXXXXXXXXX"
    data-ad-format="auto"
    data-full-width-responsive="true"></ins>
<script>(adsbygoogle = window.adsbygoogle || []).push({});</script>

4. Reklam Alanlarının Yerleşimi (Mevcut)

Aşağıdaki sayfalarda .ad-zone alanları hazır durumda beklemektedir:

Sayfa Reklam Yeri Önerilen Format
index.html Hero altı 728×90 Leaderboard
index.html Bölümler arası 336×280 Rectangle
sinif.html İçerik üstü Responsive Auto
konu.html İçerik üstü + arası Responsive Auto
test.html Test altı Responsive Auto
veli.html İçerik arası Responsive Auto

5. ads.txt Dosyası

Domain kökünde (testcoz.pro/ads.txt) aşağıdaki içeriğe sahip bir ads.txt dosyası oluşturun. Bu AdSense için zorunludur:

google.com, pub-XXXXXXXXXXXXXXXX, DIRECT, f08c47fec0942fa0

pub-XXXXXXXXXXXXXXXX yerine kendi Publisher ID'nizi yazın.

6. Soru İçerikleri Yükleme (Admin Yapısı)

Sorular js/main.js dosyasındaki SITE_DATA objesine eklenebilir. Alternatif olarak, WordPress + Elementor veya bir CMS kullanıyorsanız:

  • Custom Post Type olarak "Soru" tipi oluşturun: Sınıf / Ders / Konu / Zorluk / Soru Metni / Şıklar / Doğru Cevap
  • ACF (Advanced Custom Fields) veya CMB2 kullanarak alanları tanımlayın
  • REST API üzerinden JS ile sorgu yaparak dinamik yükleme yapın
  • Veya SQLite/JSON dosya tabanlı basit bir admin paneli tercih edin
